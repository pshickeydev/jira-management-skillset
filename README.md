# Jira Management Skillset

A set of [Agent Skills](https://agentskills.io) for managing Jira issues
via the [Atlassian Rovo MCP server](https://support.atlassian.com/atlassian-rovo-mcp-server/docs/use-atlassian-rovo-mcp-server/).
Built for use with [OpenCode](https://opencode.ai), but compatible with
any agent that supports the agentskills.io standard.

## Skills

| Skill | Command | Description |
|-------|---------|-------------|
| `jira-configure` | `/configure-jira-skillset` | Interactive setup вЂ” discover sites, projects, issue types, security levels, custom fields. Persists defaults to `config.json`. |
| `jira-create` | `/jira-create` | Create issues of any type using templates and configured defaults. |
| `jira-update` | `/jira-update` | Edit fields, update descriptions, add or modify comments. |
| `jira-transition` | `/jira-transition` | Move issues through workflow states. Discovers transitions at runtime. |
| `jira-search` | `/jira-search` | Search with JQL or natural language. Paginated results. |
| `jira-link` | `/jira-link` | Create relationships between issues (blocks, duplicates, relates). |
| `jira-bulk` | `/jira-bulk` | Batch operations вЂ” bulk close, label, reassign, or comment. |

## Getting Started

### Prerequisites

- An AI agent that supports [Agent Skills](https://agentskills.io)
  (OpenCode, Claude Code, Cursor, etc.)
- The [Atlassian Rovo MCP server](https://support.atlassian.com/atlassian-rovo-mcp-server/docs/set-up-atlassian-rovo-mcp-server/)
  configured with OAuth 2.1 authentication
- Access to at least one Jira project

### Installation

1. Clone or copy this repository.
2. Symlink the skills into your agent's skill discovery path. For
   OpenCode, symlink from `.agents/skills/`:

   ```bash
   # From your .agents/skills/ directory
   ln -s /path/to/jira-management-skillset/.agents/skills/jira-configure jira-configure
   ln -s /path/to/jira-management-skillset/.agents/skills/jira-create jira-create
   ln -s /path/to/jira-management-skillset/.agents/skills/jira-update jira-update
   ln -s /path/to/jira-management-skillset/.agents/skills/jira-transition jira-transition
   ln -s /path/to/jira-management-skillset/.agents/skills/jira-search jira-search
   ln -s /path/to/jira-management-skillset/.agents/skills/jira-link jira-link
   ln -s /path/to/jira-management-skillset/.agents/skills/jira-bulk jira-bulk
   ```

3. Optionally symlink the `/configure-jira-skillset` command:

   ```bash
   # From your .opencode/commands/ directory
   ln -s /path/to/jira-management-skillset/.opencode/commands/configure-jira-skillset.md configure-jira-skillset.md
   ```

4. Run `/configure-jira-skillset` (or ask your agent to "set up Jira")
   to create your `config.json` with site, project, and security
   defaults.

## Configuration

The `/configure-jira-skillset` command walks you through an interactive
setup wizard that:

- Discovers your accessible Jira site via the Rovo MCP
- Lets you select and configure one or more projects
- Discovers issue types, security levels, priorities, and custom fields
  from the API
- Sets defaults for priority, labels, components, assignee, and
  AI-generated comment disclaimers
- Generates issue templates in `templates/`
- Writes everything to `config.json`

Run it again at any time to add projects, update defaults, or
regenerate templates.

### `config.json`

This file is generated by `/configure-jira-skillset` and gitignored
(it contains instance-specific data). See `config.example.json` for
the schema. Key fields:

- `cloudId` вЂ” Your Jira site hostname
- `defaults` вЂ” Global defaults for project, issue type, priority,
  labels, components
- `projects.{KEY}.securityLevel` вЂ” Default issue security level
  per project
- `projects.{KEY}.commentVisibility` вЂ” Default comment restriction
  per project (role, group, or null for unrestricted)
- `assignToSelf` вЂ” Auto-assign created issues to the current user
- `aiDisclaimer` вЂ” Prepend "AI-generated" notice to all agent-posted
  content (descriptions, comments, worklogs)

## Design Principles

### Confirmation before write

Every skill shows the user exactly what will be sent to Jira and
requires explicit approval before making any write call. This applies
to issue creation, field edits, transitions, comments, and links.
Read-only operations (search, fetch) do not require confirmation.

### Runtime discovery over stale config

Dynamic values are discovered at runtime rather than read from
`config.json`, which could lose sync with the Jira instance:

- **Transitions** вЂ” Always fetched via `getTransitionsForJiraIssue`
- **Issue types** вЂ” Always fetched via `getJiraProjectIssueTypesMetadata`
- **Link types** вЂ” Always fetched via `getIssueLinkTypes`
- **Current issue state** вЂ” Always fetched via `getJiraIssue`

Static user preferences (security level, priority, labels, assignee)
are read from config.

### Versioning

Skills use [Semantic Versioning](https://semver.org/)
(`MAJOR.MINOR.PATCH`). PATCH for non-breaking, non-feature changes
(bug fixes, wording, attribution). MINOR for new capabilities. MAJOR
for breaking changes. See `AGENTS.md` for full criteria and scope
rules.

### Skill attribution

Every skill that creates or modifies Jira content appends a skill
attribution line to the body of that content (descriptions and
comments). This is unconditional вЂ” it does not depend on the
`aiDisclaimer` config option. The format is:

```
_Created with jira-create v0.1.3_
```

The AI disclaimer (when enabled) is a separate prefix at the top of
all content bodies (descriptions and comments). Attribution always
goes at the bottom.

### Data safety

- No PII or sensitive data in prompts or AI-generated content
- AI-generated content disclaimer is configurable
- Security levels and comment visibility are enforced per-project
- OAuth 2.1 only вЂ” no API tokens or passwords

## Templates

Issue templates live in `templates/` and are used by `jira-create` to
scaffold descriptions. Each template has YAML frontmatter listing
required and optional fields, followed by markdown sections.

Templates for standard types (Bug, Story, Task, Epic, Spike, Sub-task)
are included. The `/configure-jira-skillset` wizard can generate
templates for additional issue types discovered from your project.

## Authentication

This skillset uses the Atlassian Rovo MCP server with OAuth 2.1
authentication. API token authentication is not supported. If you
encounter auth errors, complete sign-in in your MCP client.
